======================================================
  다국어 TTS 벤치마크 발표 스크립트
  측정일: 2026년 3월 | 발표자: (본인)
======================================================


────────────────────────────────────────
[1] 오프닝
────────────────────────────────────────

안녕하세요. 오늘은 저희가 직접 구축한 TTS 벤치마크 환경에서
오픈소스·공개 TTS 모델 32개를 평가한 결과를 발표드리겠습니다.

평가 대상 언어는 한국어, 영어, 일본어, 중국어 총 4개 언어이고,
속도·품질·정확도·메모리 사용량을 기준으로 종합 비교했습니다.


────────────────────────────────────────
[2] 평가 개요
────────────────────────────────────────

총 32개 모델, 4개 언어, 약 4,100건 이상의 합성 결과를 분석했습니다.

평가 지표는 세 가지입니다.

첫째, RTF (Real-Time Factor) — 추론 시간을 오디오 길이로 나눈 값입니다.
1.0 미만이면 실시간 처리 가능, 낮을수록 빠릅니다.

둘째, CER/WER (Character/Word Error Rate) — Whisper large-v3로
생성된 음성을 다시 텍스트로 변환해 원문과 비교한 발음 정확도입니다.

셋째, VRAM Peak — 추론 중 최대 GPU 메모리 사용량입니다.

모델 아키텍처는 크게 세 가지로 나뉩니다.
- LLM 계열: 자기회귀 언어모델 기반. 표현력이 풍부하지만 느림 (14개 모델)
- NAR 계열: 비자기회귀. 빠르고 안정적 (12개 모델)
- Flow Matching 계열: 확산 기반의 고품질 생성 (2개 모델 + MaskGCT 등)


────────────────────────────────────────
[3] 한국어 결과
────────────────────────────────────────

한국어를 지원하는 모델은 32개 중 12개입니다.

속도 기준 순위는 다음과 같습니다.

1위: ZipVoice-FT — RTF 0.003 (저희 자체 개발 모델, REST API 기반)
2위: MeloTTS-KO  — RTF 0.045 (VITS 계열, CPU에서도 실행 가능)
3위: HierSpeech++ — RTF 0.031 (서울대 연구팀, 한국어 특화)
4위: OpenVoice-v2 — RTF 0.116 (MeloTTS + 음색 변환 2단계 파이프라인)
5위: XTTS-v2      — RTF 0.455 (17개 언어 지원, zero-shot 클론)

CER 기준으로 가장 정확한 모델은 MeloTTS-KO (CER 0%)이고,
HierSpeech++는 RTF는 빠르지만 CER 84%로 발음 정확도가 낮습니다.

주목할 점은 CosyVoice2/3, Qwen3 등 중국산 다국어 모델들은
한국어 지원을 표방하지만 실제로 레퍼런스 음성을 그대로 출력하는
"환각(hallucination)" 현상이 다수 발견됐습니다.
즉, 입력 텍스트가 아닌 레퍼런스 음성의 내용이 나오는 문제입니다.


────────────────────────────────────────
[4] 영어 결과
────────────────────────────────────────

영어는 32개 모델 중 28개가 지원하며 가장 경쟁이 치열합니다.

속도 기준 상위 모델:

1위: Kokoro-82M — RTF 0.009, MOS 4.949 (역대 최고 품질 + 최고 속도)
2위: MeloTTS-KO — RTF 0.015 (영어도 지원, CPU-friendly)
3위: OpenVoice-v2 — RTF 0.099
4위: XTTS-v2    — RTF 0.300
5위: StyleTTS2  — RTF 0.306 (VRAM 0MB, CPU 추론 가능)

Kokoro-82M은 82M 파라미터의 매우 작은 모델임에도
WV-MOS 4.949로 전체 최고 음질을 기록했습니다.
단, 사전 정의된 화자만 사용 가능하고 한국어는 미지원입니다.

Dia-1.6B는 [S1]/[S2] 두 화자 대화 형식의 독특한 모델이지만
RTF가 4.4로 매우 느려 실시간 서비스에는 부적합합니다.


────────────────────────────────────────
[5] 일본어 / 중국어 결과
────────────────────────────────────────

일본어 지원 모델은 약 8개이며,
MioTTS-1.7B (RTF 0.725), CosyVoice2 (RTF 0.562)가 상위권입니다.
다만 CosyVoice2는 일본어에서도 환각 문제가 있습니다.

중국어는 F5-TTS (RTF 0.435), Spark-TTS (RTF 1.073)가 안정적이며,
Kokoro-82M은 RTF 0.010으로 빠르지만 무한 루프 문제가 있어
신뢰도가 낮습니다.


────────────────────────────────────────
[6] 특이사항 및 문제 모델
────────────────────────────────────────

**환각(Hallucination) 문제**
- CosyVoice2/3: 전 언어에서 레퍼런스 오디오 내용 재출력
- MARS5-TTS: 일부 텍스트에서 엉뚱한 내용 생성
- Dia-1.6B: [S1]/[S2] 형식으로 인한 지표 왜곡

**속도 문제 (실시간 불가)**
- OuteTTS-0.6B: RTF 3.2 (전 언어)
- Kani-TTS-370M: 한국어 RTF 4.1
- VoiceCraft-X: 일본어 RTF 15.2 (실용 불가)
- Qwen3-TTS 계열: RTF 1.5~3.6

**한국어 지원 이슈**
- GLM-TTS: 한국어 실행 시 CUDA assertion 오류 (중국어 음소 사전 범위 초과)
  → 한국어 벤치마크에서 전면 실패
- Kokoro-82M: 한국어 미지원

**서비스 종료 모델**
- XTTS-v2 (Coqui AI 2024년 폐업): 커뮤니티 유지 보수 상태

────────────────────────────────────────
[8] 용도별 추천 모델
────────────────────────────────────────

■ 실시간 한국어 서비스
→ ZipVoice-FT (RTF 0.003) 또는 MeloTTS-KO (RTF 0.045)
  MeloTTS-KO는 CPU에서도 실시간 가능, 오픈소스 상용 가능

■ 실시간 영어 서비스
→ Kokoro-82M (RTF 0.009, MOS 4.949)
  가장 빠르고 음질도 최고. 단, 고정 화자

■ 4개 언어 동시 지원 (다국어 서비스)
→ GPT-SoVITS-V3 (RTF 0.36~0.55, 안정적)
  CosyVoice3는 다국어이나 환각 이슈로 주의 필요

■ Zero-shot 음성 클론
→ Fish-Speech-1.5 (영/일/중), XTTS-v2 (17개 언어)

■ GPU 없는 환경 (CPU 전용)
→ StyleTTS2, MeloTTS-KO, Kokoro-82M (낮은 VRAM)

■ 최고 음질 우선
→ 영어: Kokoro-82M (MOS 4.949)
  한국어: ZipVoice-FT 또는 HierSpeech++ (RTF 최고속)


────────────────────────────────────────
[9] 결론
────────────────────────────────────────

이번 벤치마크의 핵심 결론은 다음 세 가지입니다.

첫째, 속도와 품질의 트레이드오프가 여전히 존재합니다.
LLM 계열은 표현력이 좋지만 RTF 1.5~4 수준으로 실시간 서비스에
직접 사용하기 어렵고, NAR 계열은 빠르지만 표현의 다양성이 제한됩니다.

둘째, 한국어 지원의 실질적 수준이 중요합니다.
지원 언어 목록에 한국어가 있어도 실제 품질이 크게 다르며,
일부 모델은 환각 문제로 사용이 불가능합니다.

셋째, 경량 모델의 약진이 두드러집니다.
Kokoro-82M (82M 파라미터)이 영어에서 최고 품질을 기록하고,
MeloTTS-KO는 CPU에서도 실시간 한국어 합성이 가능합니다.
대형 LLM이 항상 최선은 아닙니다.

이상으로 발표를 마치겠습니다. 감사합니다.
